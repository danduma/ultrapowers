# Finishing A Development Branch

Use this skill only when the work actually happened on a branch or in a worktree that now needs a completion decision.

**Announce at start:** "I'm using the finishing-a-development-branch skill to complete this work."

## Before Offering Completion Options

- verify the relevant tests pass,
- confirm whether the current work is branch-based,
- if the work was done directly in the current repository without branch workflow, do not force this skill.

## Completion Options

When a branch exists, present clear next steps such as:

1. merge locally,
2. push and open a PR,
3. keep the branch as-is,
4. discard the branch after explicit confirmation.

If no branch-based workflow exists, simply report completion status and leave repo integration alone.
